    Lesson 1 ‚Äî Print & Output

    üß© Course 1 ‚Äî Print & Output¶

    ‚öôÔ∏è Before you start¶

    Before writing any Python code, make sure Python is installed on your machine.

    • Download Python from the official website:
      https://www.python.org/downloads/
    • Run every command yourself
    • Take your time

    Learning Python is not about speed.
    Discipline is more important than understanding everything immediately.

    üéØ What you are about to do¶

    In Lesson 1, you will:

    • write your first Python code
    • see immediate results
    • understand what the computer is doing

    You are not expected to:

    • understand everything
    • remember everything
    • be perfect

    You are expected to:

    • try
    • run the code
    • observe the output

    That‚Äôs it.

    üß† Programming is not magic¶

    Programming is often presented as something complex.

    In reality, it is very simple:

    you write instructions
    the computer follows them

    The computer does not:

    • think
    • guess
    • understand intentions

    It only follows exact instructions.

    This is good news.

    üñ®Ô∏è What is print()¶

    print() is a function.

    A function is a reusable block of code
    that performs a specific action.

    The role of print() is to: - display information for humans - show what the program is doing

    The computer executes code silently.
    print() is how you see what happens.

    ‚ñ∂Ô∏è Your first Python instruction¶

    Run this code:

    print("Hello, Python")

    What happens:

    • Python reads the instruction
    • Python executes it
    • Python displays the output

    Instruction ‚Üí execution ‚Üí output

    üß∞ Built-in functions in Python¶

    Python comes with built-in functions.

    They are:

    • already created
    • already available
    • ready to use

    Examples:

    • print()
    • len()
    • type()
    • input()

    You did not create print().
    Python provides it by default.

    üõ†Ô∏è Common uses of print()¶

    You use print() to:

    • display results
    • debug code
    • confirm actions
    • explain what a script is doing
    • make automation readable

    Automation without output is dangerous.

    üß™ Playing with output¶

    Printing text¶

    print("Network automation starts here")

    Printing numbers¶

    print(42)

    Printing multiple values¶

    print("Interface", "eth0", "is", "UP")

    üéõÔ∏è Controlling the output¶

    Separator (sep)¶

    print("eth0", "eth1", "eth2", sep=" | ")

    Output: 

    eth0 | eth1 | eth2

    Line ending (end)¶

    print("Checking interface...", end=" ")
print("DONE")

    Output: 

    Checking interface... DONE

    üåê Why output matters¶

    In networking, you always:

    • run a command
    • read the output
    • trust the result

    Programming works the same way.

    Clean output means:

    • easier debugging
    • safer automation
    • scripts others can trust

    üöÄ What matters most in this lesson¶

    At the beginning, only two things matter:

    • writing code
    • reading output

    Everything else comes later.

    Clean output is how humans trust automation.
    That is why this course starts with print().
